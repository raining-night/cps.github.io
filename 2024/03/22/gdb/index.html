gdb常用命令

Endless_daydream Lv3

基本调试

运行gdb ./xx.out启动对xx.out的调试。如果单用gdb而没有指定文件，在gdb中运行 attach xx.out 载入文件。

h(elp) [cmd]: 查看相应gdb命令的帮助信息，可以使用简写。

r(un): 进入调试后，程序不会直接开始运行，使用run指令开始运行指定的程序

q(uit): 退出调试。

k(ill): 杀死正在运行的程序，但不退出调试。可以用r重新启动程序。

源码

l(ist): 打印几行源码，反复执行会继续向下打印。

什么都不输入，直接回车，会再次执行上一条指令。

断点

如果不打断点，启动后程序会不停运行直到结束。

b(reakpoint) [line number | function]: 打断点，跟数字表示行号，跟名字表示函数名。

info b: 显示已经打的断点。每个断点有一个编号。

cl(ear) [line number | function]: 清除断点，不加参数表示清除所有。

d(elete) [breakpoint number] [ranges..]: 清除断点，用断点号(的范围)表示，不加参数表示清除所有。

en(able)/dis(able) [breakpoint number] [ranges..]: 启用/禁用断点，不清除断点。

运行

n(ext): 单步执行一行代码,如果是函数调用不会进入。

s(tep): 单步执行一行代码，会进入函数内部。

c(ontinue): 继续运行直到下一次中断，或程序结束。

fin(ish): 继续运行直到当前函数结束

u(ntil): 运行到当前循环结束

bt(breaktrace): 查看当前堆栈，多用于查看异常处。

汇编

disassemble: 打印汇编代码，对应list指令。

ni(nexti): 单步执行一行汇编(instruction),如果这一行是函数调用不会进入。

si(stepi): 单步执行一行汇编(instruction),会进入函数调用。

多窗口

layout src：显示源代码窗口

layout asm：显示汇编窗口

layout regs：显示源代码/汇编和寄存器窗口

layout split：显示源代码和汇编窗口

焦点

当焦点不在命令行窗口时，可以通过方向键来浏览所在窗口的内容，page up和page down翻页。

键盘输入仍然输入到命令行。此时如果要翻阅历史命令，用快捷键Ctrl-p和Ctrl-n替代上下键。

Ctrl-x o: 聚焦到下一个窗口

fs(focus):

update: 定位到程序正在运行的位置，用于翻页后回来。

高级调试

apropos [cmd]: 模糊匹配命令，以搜索相关命令帮助

cd: 同shell的cd，改变工作目录

pwd: 同shell，打印当前工作目录

set args：指定运行时传入的参数

show args：查看设置的参数
